# Contributing to TheGreekLab

Thank you for improving TheGreekLab. Contributions should preserve numerical
correctness, clear domain boundaries and reproducible builds.

## Development setup

You need Java 22 or newer and the included Maven wrapper. Linux CI also needs
GNU Fortran to build the native `pbivnorm` library.

Run the complete verification suite before opening a pull request:

```bash
./mvnw verify
```

On Windows PowerShell, use `./mvnw.cmd verify`.

## Making a change

1. Create a focused branch from `main`.
2. Keep production and test changes in the same pull request.
3. Add numerical reference data or cite the source of any new formula in
   [`docs/REFERENCES.md`](docs/REFERENCES.md).
4. Update `README.md`, `docs/USAGE.md` or `docs/MATH.md` when public behavior
   or mathematical assumptions change.
5. Run the complete verification suite.
6. Open a pull request and describe both the change and its validation.

Prefer small commits with imperative messages, for example
`fix: validate American option expiry`.

## Tests and numerical changes

- Cover normal cases, input validation and important boundary conditions.
- Use explicit tolerances and explain tolerances that are not self-evident.
- Do not replace an independent reference value with output generated by the
  implementation under test.
- For generated fixtures, commit the generator when possible and record its
  dependency versions, random seed, units, transformations and tolerance in
  [`docs/REFERENCES.md`](docs/REFERENCES.md).
- Keep benchmark-only code out of correctness assertions.

The build enforces at least 85% line coverage and 65% branch coverage. Coverage
is useful evidence, but it does not replace numerical cross-validation.

## Licensing

By submitting a contribution, you agree that it may be distributed under the
component-specific terms described in [`LICENSING.md`](LICENSING.md). Do not
add third-party code or data without recording its license and provenance.

## Reporting vulnerabilities

Do not disclose security vulnerabilities in a public issue. Follow the private
reporting process in [`SECURITY.md`](SECURITY.md).
